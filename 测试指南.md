# 🧪 DuckDuckGo + TEmail 测试指南

## 快速测试（5 分钟）

### 第 1 步：重新加载扩展
1. 打开 `chrome://extensions/`
2. 找到你的扩展
3. 点击刷新按钮 🔄

### 第 2 步：配置扩展
1. 点击扩展图标
2. 选择 "DuckDuckGo" 模式
3. 输入你的 DuckDuckGo Token
4. 点击 "自动验证码配置（可选）" 展开
5. 输入以下信息：
   - TEmail 邮箱地址: `dkdkgo@chaijz.top`（默认已填写）
   - JWT Token 或 Admin 密码（二选一）
6. 点击 "保存配置"

### 第 3 步：打开控制台
1. 按 F12 打开开发者工具
2. 切换到 "Console" 标签
3. 清空控制台（右键 → Clear console）

### 第 4 步：开始测试
1. 在扩展 Popup 中设置：
   - 注册数量: 1
   - 并发数: 1
2. 点击 "开始注册"
3. 观察控制台日志

## ❌ 常见错误

### 错误 1: 邮箱地址不完整
```
❌ [DuckDuckGo+TEmail] ✓ 别名已创建: resume-sage-snub
```

**应该是**:
```
✅ [DuckDuckGo+TEmail] ✓ 别名已创建: resume-sage-snub@duck.com
```

**解决方案**: 已修复，如果仍然出现请报告

### 错误 2: 跳过新邮件
```
❌ [TEmail]   ✗ 跳过旧邮件 (时间早于起始时间)
```

**应该是**:
```
✅ [TEmail]   ✓ 邮件 ID 符合条件 (101 > 100)
```

**解决方案**: 已修复（使用邮件 ID 代替时间），如果仍然出现请报告

### 错误 3: 找不到验证码
```
❌ [TEmail] 尝试 30/30...
❌ 超时：未能在规定时间内收到验证码
```

**可能原因**:
1. DuckDuckGo 转发延迟（等待更长时间）
2. TEmail 配置错误（检查邮箱地址和认证信息）
3. 邮件被过滤（检查 TEmail 收件箱）

**检查步骤**:
1. 手动登录 TEmail (`https://email.chaijz.top`)
2. 查看是否收到来自 AWS 的邮件
3. 如果收到了但扩展没找到，复制日志报告问题

### 错误 4: JWT Token 获取失败
```
❌ [TEmail] 获取 JWT 失败: 401
```

**解决方案**:
1. 检查 Admin 密码是否正确
2. 或者直接使用 JWT Token（推荐）

## 🐛 如何报告问题

如果测试失败，请提供：

1. **完整的控制台日志**
   - 从 `[Service Worker] 开始批量注册` 开始
   - 到错误信息结束
   - 复制所有 `[DuckDuckGo]` 和 `[TEmail]` 开头的日志

2. **配置信息**（隐藏敏感信息）
   ```
   - DuckDuckGo Token: 已配置 ✅
   - TEmail 邮箱: dkdkgo@chaijz.top
   - 认证方式: JWT Token / Admin 密码
   ```

3. **问题描述**
   - 在哪一步失败？
   - 错误信息是什么？
   - 是否能手动收到邮件？

## 💡 调试技巧

### 技巧 1: 检查 TEmail 是否收到邮件
1. 打开 `https://email.chaijz.top`
2. 使用你的 TEmail 账号登录
3. 查看收件箱是否有来自 AWS 的邮件
4. 如果有邮件但扩展没找到，说明过滤逻辑有问题

### 技巧 2: 检查邮件 ID
在 TEmail 收件箱中，查看邮件的 ID（通常在 URL 或邮件详情中）：
- 如果新邮件的 ID 确实大于起始 ID，但被跳过了 → 报告 Bug
- 如果新邮件的 ID 小于或等于起始 ID → 可能是 TEmail 的 ID 分配问题

### 技巧 3: 增加日志
如果需要更详细的日志，可以在代码中添加：
```javascript
console.log('[DEBUG] 邮件详情:', mail);
```
